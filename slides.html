Table stakes: the gaps that end conversations immediately
If you don't have these, the evaluation stops before it starts.
01
No environment separation
Production, staging, and dev run in the same place; buyers won't touch it.
02
No audit trail
Every action, every access, every model decision needs to be logged and retrievable.
03
Shared credentials & API keys
One set of credentials passed around Slack. No secrets management, no key rotation, no individual access.
04
Missing role-based access
No way to limit what different users can see or do; fails every security review.
05
No versioning or rollback (app + model)
Can't revert when something goes wrong. Can't trace which model version produced which output.
The gaps that kill deals after the pilot succeeds
These are the ones founders don't see coming. Each showed up in a real deal that stalled or died.
06. No data deletion or retention mechanism.
Legal asks "can you purge a patient's data on request?" and your engineer says "we'd have to write a script."
07. Hard-coded integrations
Built for one EHR or one data format; breaks when the next buyer uses something different.
08. Manual deployments to production.
No CI/CD pipeline, someone SSHs into a server to push changes. One bad deploy with no rollback path and you're down in production.
09. No tenant-level configuration
Changing a threshold or workflow rule requires a full deploy instead of a config publish. Engineering becomes a bottleneck for every customer request.
10. Monitoring exists, nobody watches it
"What happens when your system goes down at 2am?" Honest answer: someone notices in the morning.
11. No model transparency documentation
The CMO asks "why did it flag this patient, and was it tested on a population like ours?" You hand them a research paper instead of a model card with explainability methods, training data provenance, and bias testing.
The questions that come up in every Series A+ diligence
Technical diligence isn't about your tech stack. It's about whether your decisions signal a company that can scale, or a science project.
"What breaks at 10x, 100x your current load?"
Tests whether you understand your own bottlenecks.
"How long to onboard a new enterprise customer?"
Reveals operational maturity and multi-tenancy design.
"What's your bus factor on critical systems?"
Team depth and documentation practices.
"Walk me through your last incident."
Culture of accountability and operational readiness.
"How was your model validated, and on whose data?"
Bias risk, generalizability, and whether clinical performance holds across populations.
Triage: what to fix now vs. what can wait
You have 12 months of runway and 10 engineers. You can't rebuild everything. Prioritize.
Fix this quarter
Expensive to retrofit. Will block deals.
  • Audit trail
  • Role-based access
  • Environment separation
  • Data deletion/retention policy
  • Credential management
  • Data residency documentation (where data lives, who can access it, can it stay in-region)
  • Versioning + rollback
  • Manual deployments
  • Model transparency: model card, training data provenance
Plan for future
Important, but can be phased in.
  • Integration abstraction layer
  • Tenant-level configuration
  • Monitoring + alerting + on-call
  • Model transparency: bias testing
  • SOC 2 Type II
  • Disaster recovery testing
  • API sandbox
  • SSO / SAML
